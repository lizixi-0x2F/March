Trie-based KV cache with prefix sharing for LLM inference.
Store each prefix once. Serve it to everyone.

The problem with KV cache today

Every LLM inference request that shares a system prompt or conversation history recomputes — or re-stores — the same key/value tensors. With 100 concurrent sessions sharing a 512-token system prompt, you're keeping 100 identical copies in GPU memory. That's not a rounding error; it's the dominant cost.

The prefix is the same. The memory isn't.

Existing solutions either require custom inference engines, invasive model changes, or give up on exact sharing and approximate instead. March does none of that.

Prefix sharing via a token Trie

March inserts token sequences into a Trie. Each node in the Trie corresponds to one token. When two sequences share a prefix, they share the same path through the Trie — and therefore the same physical memory pages. No copies. No indirection. 

# Two sequences, one shared prefix
insert([sys, A, B, C],    kv₁)
insert([sys, A, B, C, D], kv₂)

  root
  └── [sys] → [A] → [B] → [C] ← KVPage (shared)
                                └── [D] ← KVPage (new, 1 page only)

query([sys, A, B, C, D])
  → walks 5 hops, each O(1) via internal hashmap
  → returns zero-copy pointers: [page_C, page_D]
  → total: O(L), zero allocations, zero memcpy

Physical memory lives in a fixed-size mmap'd pool. march_query returns raw pointers directly into that pool — the inference engine reads KV data without a single byte being copied.

Requires: gcc, Python ≥ 3.10. HuggingFace integration requires transformers ≥ 4.40 and torch.

Usage

Core cache

from march import MarchCache

cache = MarchCache(page_size=4096, max_pages=256)

cache.insert([1, 2, 3], kv_bytes)
pages, matched = cache.query([1, 2, 3, 4])

cache.print_stats()
# ── March Stats ──────────────────────────
#   hit rate    : 100.0%  (1/1)
#   token reuse : 75.0%   (3/4 tokens from cache)
#   page pool   : 256 pages × 4096 B = 1024.0 KB
# ─────────────────────────────────────────

HuggingFace integration

from march.hf import MarchPrefixCache

prefix_cache = MarchPrefixCache(model, tokenizer)

# Miss — runs prefill, stores result
past_kv, hit = prefix_cache.prefill(input_ids)   # hit=False

# Hit — zero forward pass, served from Trie
past_kv, hit = prefix_cache.prefill(input_ids)   # hit=True

outputs = model.generate(
    input_ids,
    past_key_values=past_kv,
    use_cache=True,
)

Architecture

Three C modules, one Python binding layer, no runtime dependencies.

PageAllocator

mmap-backed fixed-size pool. Alloc and free are O(1) via a free-list stack. Pages are reference-counted for safe sharing. LRU timestamps enable eviction when the pool fills.

KVTrie

Token-keyed prefix trie. Each node holds an open-addressing hashmap (Fibonacci hashing, linear probing, load factor ≤ 0.5) over its children, so each hop is O(1) regardless of vocabulary size. Nodes without KV data act as internal routing nodes.

ViewBuilder

Walks a path through the Trie and fills a caller-provided pointer array with references to live pool pages — root to leaf order, zero-copy. The caller's inference engine reads directly from the mmap'd buffer.

March is early-stage research software. The C core is stable; the Python API may evolve. Contributions welcome.
